# 错误处理

## 核心：Result 类型

本项目使用 `neverthrow` 库进行错误处理。底层函数（API 层、工具函数等）返回 `Result<R, E>` 类型。

**预定义类型**（在 `src/types/lib.ts`）：

```typescript
import type { Result } from 'neverthrow'

export type SafeAsync<T> = Promise<Result<T, Error>>
export type Safe<T> = Result<T, Error>

export type TryCMD = {
  (...args: Parameters<typeof invoke>): SafeAsync<undefined>
  <T>(...args: Parameters<typeof invoke>): SafeAsync<T>
}
```

## 错误处理工具

在 `src/lib/index.ts` 中已定义以下工具函数：

```typescript
// 同步函数包装
const safeFn = tryResult(unsafeFunction)

// 异步函数包装
const safeAsyncFn = tryResultAsync(asyncFunction)

// JSON 解析
const parsed = parseJson(jsonString)

// Tauri 命令调用
const result = await tryCMD<T>('command_name', { param: value })
```

## Result 操作模式

**Result 从底层向上传递**，使用 `map`、`mapErr` 进行转换，使用 `match` 解包：

```typescript
// ✅ 使用 map/mapErr 链式操作
const result = await createSession(data)
  .map((session) => ({ ...session, extraField }))
  .mapErr((e) => new Error(`创建会话失败: ${e.message}`))

// ✅ 使用 match 解包
result.match(
  (session) => {
    setCurrentSession(session)
  },
  (error) => {
    toast.error(error.message)
  }
)

// ⚠️ 允许：必要时使用 isOk/isErr 解包
if (result.isOk()) {
  const session = result.value
  setCurrentSession(session)
} else {
  toast.error(result.error.message)
}
```

## API 层错误处理

所有 API 命令函数使用 `tryCMD` 包装，返回 `SafeAsync<T>`：

```typescript
export const createProduct = (data: CreateProductDto) =>
  tryCMD<Product>('create_product', { input: data })
```

## 页面组件中的错误处理

在页面组件中使用 `result.match()` 处理成功和失败：

```typescript
const loadData = useCallback(async () => {
  setLoading(true)
  const result = await productApi.getAll()
  result.match(
    (data) => {
      setProducts(data)
      setLoading(false)
    },
    (error) => {
      toast.error(`加载失败: ${error.message}`)
      setLoading(false)
    }
  )
}, [])
```

## AI Hooks 中的错误处理

AI 相关 Hooks（`useSessionList`、`useSectionList`、`useSectionChat`）使用 `try-finally` 和直接 `try-catch` 模式，不使用 `SafeAsync` 包装：

```typescript
// useSessionList 模式：try-finally 确保 loading 状态重置
const loadTodayLastSession = useCallback(async () => {
  setIsLoading(true)
  try {
    const allSessions = await getAllSessions()
    setSessions(allSessions)
    // ...
  } finally {
    setIsLoading(false)
  }
}, [])

// useSectionChat 模式：try-catch 捕获错误
const send = useCallback(
  async (content: string) => {
    try {
      setIsStreaming(true)
      setError(null)
      // ... Agent 调用和流式处理
    } catch (e) {
      setError(e instanceof Error ? e.message : '发生未知错误')
    } finally {
      setIsStreaming(false)
    }
  },
  [
    /* deps */
  ]
)
```

这种差异的原因是：AI Hooks 直接调用存储层函数（`@/ai/storage/`），这些函数不返回 `Result` 类型；而普通业务页面调用 API 层（`@/api/commands/`），API 层统一使用 `tryCMD` 返回 `Result`。
