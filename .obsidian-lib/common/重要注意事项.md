# 重要注意事项

## 开发环境

- **CSP 策略**：开发模式下 CSP 已禁用（tauri.conf.json 配置）
- **开发服务器**：Vite 运行在端口 1420（tauri.conf.json 配置）
- **构建顺序**：前端必须先于 Tauri 打包构建（由 `beforeBuildCommand` 自动处理）

## 数据库

- **存储位置**：SQLite 数据库文件存储在应用数据目录中
- **Entity-First**：实体注册必须在数据库初始化之后执行
- **实体自动同步**：Sea-ORM 实体定义在 Rust 代码中先行编写，数据库 schema 在应用启动时自动创建/更新

## 金融精度

- **所有金额字段必须使用 `rust_decimal::Decimal`**，严禁使用浮点数
- 前端传输使用字符串序列化，避免 JSON 数字精度丢失

## 路径别名

- 前端组件导入统一使用 `@/` 前缀（在 components.json 中配置）
- 禁止使用相对路径跨目录引用

## Tauri 应用初始化流程

应用启动时按以下顺序执行（参见 `src-tauri/src/lib.rs`）：

1. 创建应用数据目录
2. 初始化 Tokio 运行时
3. 初始化 SQLite 数据库连接池
4. 注册 Sea-ORM 实体并同步 schema
5. 初始化服务层
6. 安装 Tauri 命令
7. 将数据库连接纳入应用状态管理

## 开发命令

### 完整应用

```bash
pnpm tauri dev          # 运行桌面应用（推荐，支持热重载）
pnpm tauri build        # 构建原生桌面应用
```

### 仅前端

```bash
pnpm dev                # 启动 Vite 开发服务器（端口 1420）
pnpm build              # 构建前端（先执行 tsc 类型检查）
pnpm preview            # 预览构建后的前端
```

### 后端（Rust）

```bash
cd src-tauri
cargo build             # 构建 Rust 后端
cargo run               # 独立运行 Rust 后端
cargo test              # 运行 Rust 测试
cargo clippy            # 运行 Rust 代码检查
```
