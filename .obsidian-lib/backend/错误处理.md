# 错误处理

## 错误处理层级

项目采用分层的错误处理策略，各层使用不同的错误类型：

```
服务层: Result<T, Box<dyn std::error::Error>>
    ↓ .map_err(|e| e.to_string())
命令层: Result<T, String>
    ↓ JSON 序列化
前端: Error 对象
```

## 服务层错误处理

所有服务统一使用 `Box<dyn std::error::Error>` 作为错误类型：

```rust
// ✅ 正确：服务层统一错误类型
pub async fn create_customer(
    &self,
    input: CreateCustomerDto,
) -> Result<Model, Box<dyn std::error::Error>> {
    let id = Model::generate_id(&self.db).await?;
    let record = Entity::find_by_id(id)
        .one(&self.db)
        .await?
        .ok_or("客户不存在")?;
    // ...
}
```

## 命令层错误转换

命令层将服务层错误统一转换为 `String`：

```rust
#[tauri::command]
pub async fn create_customer(
    service: State<'_, CustomerService>,
    input: CreateCustomerDto,
) -> Result<Model, String> {
    service.create_customer(input)
        .await
        .map_err(|e| e.to_string())
}
```

## 错误转换方式

### 使用 `?` 操作符传播错误

```rust
// 自动传播 Box<dyn Error>
let record = Entity::find_by_id(id)
    .one(&self.db)
    .await?;
```

### 使用 `map_err` 转换错误信息

```rust
let parsed_datetime = NaiveDateTime::parse_from_str(&self.record_time, "%Y-%m-%d %H:%M:%S")
    .map_err(|_| "日期格式无效，期望 YYYY-MM-DD HH:MM:SS".to_string())?;
```

### 使用 `ok_or` / `ok_or_else` 转换 Option

```rust
let record = Entity::find_by_id(id)
    .one(&self.db)
    .await?
    .ok_or("记录不存在")?;

// 或带上下文信息
let record = Entity::find_by_id(id)
    .one(&self.db)
    .await?
    .ok_or_else(|| format!("记录不存在: ID={}", id))?;
```

### 使用 `.into()` 创建简单错误

```rust
return Err("只有待入账状态的记录可以修改".into());
```

## 常见错误处理模式

### 参数验证

```rust
pub async fn create_record(
    &self,
    input: CreateRecordDto,
) -> Result<Model, Box<dyn std::error::Error>> {
    if input.amount <= Decimal::ZERO {
        return Err("金额必须大于零".into());
    }

    if input.title.is_empty() {
        return Err("标题不能为空".into());
    }

    // ...
}
```

### 业务规则验证

```rust
pub async fn modify_record(&self, input: ModifyDto) -> Result<Model, Box<dyn std::error::Error>> {
    // 获取现有记录
    let record = Entity::find_by_id(input.id)
        .one(&self.db)
        .await?
        .ok_or("记录不存在")?;

    // 验证业务规则
    if record.state != AccountingRecordState::PendingPosting {
        return Err("只有待入账状态的记录可以修改".into());
    }

    // 执行更新
    // ...
}
```

### 删除操作返回 bool

```rust
#[tauri::command]
pub async fn delete_customer(
    service: State<'_, CustomerService>,
    id: i64,
) -> Result<bool, String> {
    service.delete_customer(id)
        .await
        .map(|_| true)
        .map_err(|e| e.to_string())
}
```

## 避免的做法

```rust
// ❌ 避免 unwrap（生产代码中）
let result = operation().unwrap();

// ❌ 避免 panic
if condition {
    panic!("不应该发生");
}

// ✅ 正确做法：使用 Result 和 ? 传播
let result = operation()?;

// ✅ 正确做法：初始化阶段可用 expect
let runtime = tokio::runtime::Runtime::new()
    .expect("创建运行时失败");
```
