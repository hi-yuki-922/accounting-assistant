# 服务层 (Services)

## 服务模块结构

每个服务模块统一使用以下结构：

```
services/
├── [模块名称]/
│   ├── mod.rs      # 导出服务实现和 DTO
│   ├── service.rs  # 具体服务实现
│   └── dto.rs      # DTO 定义
```

### mod.rs 模板

```rust
// services/customer/mod.rs
pub mod dto;
pub mod service;

pub use service::CustomerService;
pub use dto::*;
```

### service.rs 模板

```rust
// services/customer/service.rs
use sea_orm::DatabaseConnection;

use crate::entity::customer;
use super::dto::*;

/// 客户服务
#[derive(Debug)]
pub struct CustomerService {
    db: DatabaseConnection,
}

impl CustomerService {
    pub fn new(db: DatabaseConnection) -> Self {
        Self { db }
    }

    /// 创建客户
    pub async fn create_customer(
        &self,
        input: CreateCustomerDto,
    ) -> Result<customer::Model, Box<dyn std::error::Error>> {
        let id = customer::Model::generate_id(&self.db).await?;
        // ...
    }

    /// 获取所有客户
    pub async fn get_all_customers(
        &self,
    ) -> Result<Vec<customer::Model>, Box<dyn std::error::Error>> {
        customer::Entity::find()
            .order_by_desc(customer::Column::CreateAt)
            .all(&self.db)
            .await
            .map_err(|e| e.into())
    }
}
```

### dto.rs 模板

```rust
// services/customer/dto.rs
use serde::{Deserialize, Serialize};

/// 创建客户 DTO
#[derive(Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct CreateCustomerDto {
    pub name: String,
    pub category: String,
    pub phone: String,              // 必填
    pub wechat: Option<String>,
    pub address: Option<String>,
    pub bank_account: Option<String>,
    pub remark: Option<String>,
}

/// 更新客户 DTO
#[derive(Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct UpdateCustomerDto {
    pub id: i64,
    pub name: Option<String>,
    pub category: Option<String>,
    pub phone: Option<String>,
    pub wechat: Option<Option<String>>,
    pub address: Option<Option<String>>,
    pub bank_account: Option<Option<String>>,
    pub remark: Option<Option<String>>,
}
```

## 服务结构

每个服务持有数据库连接：

```rust
#[derive(Debug)]
pub struct XxxService {
    db: DatabaseConnection,
}

impl XxxService {
    pub fn new(db: DatabaseConnection) -> Self {
        Self { db }
    }
}
```

> **注意**：`ChatService` 额外派生了 `Clone`。

## 方法命名

### CRUD 操作标准命名

| 操作     | 方法名模式        | 示例                                                |
| -------- | ----------------- | --------------------------------------------------- |
| 创建     | `create_xxx`      | `create_customer`, `create_order`                   |
| 查询全部 | `get_all_xxxs`    | `get_all_customers`, `get_all_products`             |
| 查询单个 | `get_xxx_by_id`   | `get_customer_by_id`                                |
| 搜索     | `search_xxxs`     | `search_customers`, `search_products`               |
| 更新     | `update_xxx`      | `update_customer`, `update_product`                 |
| 删除     | `delete_xxx`      | `delete_customer`                                   |
| 条件查询 | `get_xxxs_by_yyy` | `get_orders_by_status`, `get_orders_by_customer_id` |
| 复合查询 | `query_xxxs`      | `query_orders`                                      |

### 方法签名

```rust
// 创建 — 返回创建的实体
pub async fn create_customer(&self, input: CreateCustomerDto) -> Result<Model, Box<dyn std::error::Error>>

// 查询全部 — 返回列表
pub async fn get_all_customers(&self) -> Result<Vec<Model>, Box<dyn std::error::Error>>

// 查询单个 — 返回单个实体
pub async fn get_customer_by_id(&self, id: i64) -> Result<Model, Box<dyn std::error::Error>>

// 搜索 — 返回匹配列表
pub async fn search_customers(&self, keyword: String) -> Result<Vec<Model>, Box<dyn std::error::Error>>

// 更新 — 返回更新后的实体
pub async fn update_customer(&self, input: UpdateCustomerDto) -> Result<Model, Box<dyn std::error::Error>>

// 删除 — 返回空
pub async fn delete_customer(&self, id: i64) -> Result<(), Box<dyn std::error::Error>>
```

## 业务规则验证

```rust
pub async fn update_record(&self, input: ModifyDto) -> Result<Model, Box<dyn std::error::Error>> {
    // 获取现有记录
    let record = Entity::find_by_id(input.id)
        .one(&self.db)
        .await?
        .ok_or("记录不存在")?;

    // 验证业务规则：只有待入账状态可修改
    if record.state != AccountingRecordState::PendingPosting {
        return Err("只有待入账状态的记录可以修改".into());
    }

    // 执行更新...
}
```

## 各服务方法一览

### AccountingService（记账服务）

| 方法                      | 说明                                     |
| ------------------------- | ---------------------------------------- |
| `create_record`           | 创建记账记录，更新账本 record_count      |
| `update_record`           | 更新记账记录（仅待入账状态可修改）       |
| `post_record`             | 过账（将状态改为 Posted）                |
| `delete_record`           | 删除记账记录（仅限待入账且无冲账关联）   |
| `batch_post_records`      | 批量入账（预验证 + 事务保证原子性）      |
| `create_write_off_record` | 创建冲账记录（已入账记录才能冲账）       |
| `get_records_by_order_id` | 根据订单 ID 查询关联的记账记录（含冲账） |

### AccountingBookService（账本服务）

| 方法                               | 说明                                     |
| ---------------------------------- | ---------------------------------------- |
| `create_default_book`              | 创建默认账本（未归类账目）               |
| `create_book`                      | 创建账本                                 |
| `get_all_books`                    | 查询所有账本                             |
| `get_book_by_id`                   | 根据ID查询单个账本                       |
| `update_book`                      | 更新账本信息                             |
| `delete_book`                      | 删除账本（关联记录迁移到默认账本）       |
| `get_records_by_book_id`           | 查询指定账本下的所有记录                 |
| `get_uncategorized_records`        | 查询未归类账目（NULL 和默认账本的记录）  |
| `get_books_paginated`              | 分页查询账本列表                         |
| `get_records_by_book_id_paginated` | 分页查询指定账本下的记账记录             |
| `get_write_off_records_by_id`      | 根据记录 ID 查询冲账关联记录             |
| `get_record_write_off_details`     | 查询记录的冲账详情（HoverCard 按需加载） |

### CategoryService（品类服务）

```rust
/// 默认品类名称
pub const DEFAULT_CATEGORY_NAME: &str = "未分类";
```

| 方法                      | 说明                                               |
| ------------------------- | -------------------------------------------------- |
| `create_category`         | 创建品类（验证名称唯一、销售/进货账本存在）        |
| `update_category`         | 更新品类（"未分类"名称不可修改，名称唯一性校验）   |
| `delete_category`         | 删除品类（"未分类"不可删除，有商品关联时不可删除） |
| `get_all_categories`      | 查询所有品类（"未分类"排首位，其余按创建时间升序） |
| `get_category_by_id`      | 根据 ID 查询品类                                   |
| `create_default_category` | 创建默认品类（系统启动时调用）                     |

### ChatService（聊天服务）

> **注意**：`ChatService` 额外派生了 `Clone`。

| 方法                       | 说明                                                          |
| -------------------------- | ------------------------------------------------------------- |
| `create_session`           | 创建新会话                                                    |
| `get_all_sessions`         | 获取所有会话（按创建时间倒序）                                |
| `get_session_by_id`        | 根据 ID 获取会话                                              |

### OrderService（订单服务）

| 方法                        | 说明                                                  |
| --------------------------- | ----------------------------------------------------- |
| `create_order`              | 创建订单（事务中创建订单和明细，生成编号）            |
| `settle_order`              | 结账订单（按品类分组记账 + 折扣冲账）                 |
| `get_settle_preview`        | 获取结算预览（按品类分组展示记账预览 + 折扣冲账预览） |
| `cancel_order`              | 取消订单（仅 Pending 状态可取消）                     |
| `update_order`              | 编辑订单（仅 Pending 状态可编辑）                     |
| `get_all_orders`            | 查询所有订单（按创建时间倒序）                        |
| `get_order_by_id`           | 根据 ID 查询订单（含明细）                            |
| `get_orders_by_customer_id` | 按客户查询订单                                        |
| `get_orders_by_status`      | 按状态筛选订单                                        |
| `query_orders`              | 分页查询订单（支持多维度筛选）                        |

> **注意**：`settle_order` 接受 `SettleOrderDto` 参数（含 `order_id`、`channel`、`actual_amount`），而非仅 `order_id`。

## 跨模块操作

当服务需要操作其他模块的数据时（如订单结算创建记账记录），直接通过数据库连接操作对应实体：

```rust
// OrderService.settle_order() 示例
pub async fn settle_order(&self, input: SettleOrderDto) -> Result<OrderModel, Box<dyn std::error::Error>> {
    let txn = self.db.begin().await?;

    // 更新订单状态
    // ...

    // 按品类分组创建关联的记账记录
    // ...

    // 创建折扣冲账记录（如有折扣）
    // ...

    // 更新账本记录计数
    // ...

    txn.commit().await?;
}
```
