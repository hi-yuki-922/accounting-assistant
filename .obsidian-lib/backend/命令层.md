# 命令层 (Commands)

## 命令定义

统一使用 `#[tauri::command]` 宏定义 Tauri 命令：

```rust
use tauri::State;

use crate::services::CustomerService;
use crate::entity::customer;
use super::dto::*;

/// 创建客户
#[tauri::command]
pub async fn create_customer(
    service: State<'_, CustomerService>,
    input: CreateCustomerDto,
) -> Result<customer::Model, String> {
    service.create_customer(input)
        .await
        .map_err(|e| e.to_string())
}
```

> **规范**：所有命令必须使用 `#[tauri::command]` 全限定注解，不使用 `#[command]` 短写法。

## 状态注入

### 服务注入

通过 `State<'_, ServiceType>` 注入服务实例：

```rust
#[tauri::command]
pub async fn get_all_products(
    service: State<'_, ProductService>,
) -> Result<Vec<product::Model>, String> {
    service.get_all_products()
        .await
        .map_err(|e| e.to_string())
}
```

### AppHandle 注入

需要访问应用资源时注入 `AppHandle`（如文件操作）：

```rust
use tauri::AppHandle;

#[tauri::command]
pub async fn create_attachment(
    app: AppHandle,
    service: State<'_, AttachmentService>,
    master_id: i64,
    file_name: String,
    file_suffix: String,
    file_size: String,
    file_content: Vec<u8>,
) -> Result<(i64, String), String> {
    service.create_attachment(&app, master_id, file_name, file_suffix, file_size, file_content)
        .await
        .map_err(|e| e.to_string())
}
```

## 参数处理

### 基本类型

```rust
#[tauri::command]
pub async fn delete_customer(
    service: State<'_, CustomerService>,
    id: i64,
) -> Result<bool, String> {
    service.delete_customer(id)
        .await
        .map(|_| true)
        .map_err(|e| e.to_string())
}
```

### DTO 类型

```rust
#[tauri::command]
pub async fn create_order(
    service: State<'_, OrderService>,
    input: CreateOrderDto,
) -> Result<OrderDetail, String> {
    service.create_order(input)
        .await
        .map_err(|e| e.to_string())
}
```

### 必填参数

```rust
#[tauri::command]
pub async fn search_customers(
    service: State<'_, CustomerService>,
    keyword: String,
) -> Result<Vec<customer::Model>, String> {
    service.search_customers(keyword)
        .await
        .map_err(|e| e.to_string())
}
```

### 实体 Model

```rust

    service.get_customer_by_id(id)
        .await
        .map_err(|e| e.to_string())

```

### 列表

```rust
#[tauri::command]
pub async fn get_all_customers(
    service: State<'_, CustomerService>,
) -> Result<Vec<customer::Model>, String> {
    // ...
}
```

### 删除操作返回 bool

```rust
#[tauri::command]
pub async fn delete_customer(
    service: State<'_, CustomerService>,
    id: i64,
) -> Result<bool, String> {
    service.delete_customer(id)
        .await
        .map(|_| true)
        .map_err(|e| e.to_string())
}
```

### 删除操作返回 u64

```rust
// delete_chat_session 返回受影响的行数
#[tauri::command]
pub async fn delete_chat_session(
    service: State<'_, ChatService>,
    id: i64,
) -> Result<u64, String> {
    service.delete_session(id)
        .await
        .map_err(|e| e.to_string())
        .map(|result| result.rows_affected)
}
```

### 自定义响应结构

对于需要聚合数据的场景，在命令文件中定义局部响应结构：

```rust
// commands/order.rs
use serde::Serialize;

/// 订单详情返回类型（订单 + 明细列表）
#[derive(Serialize)]
#[serde(rename_all = "camelCase")]
pub struct OrderDetail {
    pub order: OrderModel,
    pub items: Vec<OrderItemModel>,
}

/// 分页查询结果返回类型
#[derive(Serialize)]
#[serde(rename_all = "camelCase")]
pub struct QueryOrdersResult {
    pub orders: Vec<OrderModel>,
    pub total: u64,
}

#[tauri::command]
pub async fn get_order_detail(
    service: State<'_, OrderService>,
    id: i64,
) -> Result<Option<OrderDetail>, String> {
    service.get_order_by_id(id)
        .await
        .map(|opt| opt.map(|(order, items)| OrderDetail { order, items }))
        .map_err(|e| e.to_string())
}
```

## 命名约定

命令函数名使用 snake_case，与前端调用保持一致：

```rust
// Rust 命令定义
#[tauri::command]
pub async fn create_customer(...) { }

// 前端调用
invoke('create_customer', { input: data })
```

## 错误处理

命令层统一将服务层的 `Box<dyn std::error::Error>` 转换为 `String`：

```rust
service.method(params)
    .await
    .map_err(|e| e.to_string())
```

删除操作额外映射为 `bool`：

```rust
service.delete_xxx(id)
    .await
    .map(|_| true)
    .map_err(|e| e.to_string())
```
