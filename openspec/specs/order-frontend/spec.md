# 订单管理前端（order-frontend）

## Purpose

提供订单管理的前端用户界面，包括卡片/列表双视图、订单详情 Dialog、创建/编辑/结账/取消等对话框交互流程。

## Requirements

### Requirement: 订单管理路由
系统 SHALL 提供 `/orders` 路由，使用 `AppLayout` 布局，包含订单管理页面（不包含独立详情页路由）。

#### Scenario: 访问订单管理
- **WHEN** 用户点击侧边栏"订单管理"导航项
- **THEN** 系统导航到 `/orders`，展示订单管理页面

### Requirement: 卡片/列表双视图切换
系统 SHALL 在订单管理页面提供视图切换控件，支持卡片视图和列表视图两种模式。

#### Scenario: 切换到卡片视图
- **WHEN** 用户点击"卡片视图"切换按钮
- **THEN** 页面切换为卡片视图，仅展示今日订单，显示类型 Tab 和状态 Tab

#### Scenario: 切换到列表视图
- **WHEN** 用户点击"列表视图"切换按钮
- **THEN** 页面切换为列表视图，展示全部历史订单，显示筛选栏和分页

### Requirement: 卡片视图展示
系统 SHALL 在卡片视图中仅展示今日订单，支持按类型和状态双 Tab 筛选。

#### Scenario: 类型 Tab 筛选
- **WHEN** 用户切换类型 Tab（全部/销售订单/采购订单）
- **THEN** 卡片列表仅显示对应类型的今日订单

#### Scenario: 状态 Tab 筛选
- **WHEN** 用户切换状态 Tab（全部/待结账/已结账/已取消）
- **THEN** 卡片列表仅显示对应状态的今日订单

#### Scenario: 类型 + 状态组合筛选
- **WHEN** 用户同时选择类型"销售订单"和状态"待结账"
- **THEN** 卡片列表仅显示今日的待结账销售订单

#### Scenario: 显示订单卡片
- **WHEN** 卡片视图加载完成
- **THEN** 每个订单以卡片形式展示，包含订单编号（#N）、类型（销售/采购）、客户名称（或"散客"）、应收金额、实收金额、状态、创建时间

#### Scenario: 空状态展示
- **WHEN** 当前筛选条件下没有订单
- **THEN** 显示空状态提示

### Requirement: 列表视图展示
系统 SHALL 在列表视图中以表格形式展示订单，支持多维度筛选和分页。

#### Scenario: 显示订单列表
- **WHEN** 列表视图加载完成
- **THEN** 每行显示：订单编号、类型、客户、应收金额、实收金额、状态、支付渠道、创建时间

#### Scenario: 多维度筛选
- **WHEN** 用户使用筛选栏设置条件（时间范围、订单状态、金额范围、支付渠道、订单类型）
- **THEN** 列表仅显示满足所有筛选条件的订单

#### Scenario: 分页浏览
- **WHEN** 订单数量超过每页条数
- **THEN** 列表底部分页控件，支持翻页浏览

#### Scenario: 点击行查看详情
- **WHEN** 用户点击列表中的某一行
- **THEN** 弹出订单详情 Dialog，显示订单完整信息和明细

### Requirement: 订单详情 Dialog
系统 SHALL 提供订单详情 Dialog，展示订单完整信息、明细列表和操作按钮。不使用独立页面。

#### Scenario: 查看订单详情
- **WHEN** 用户点击订单卡片或列表行
- **THEN** 弹出 Dialog 展示订单基本信息（编号、类型、客户、金额、状态、渠道、备注、创建时间）
- **THEN** Dialog 展示订单明细表格（商品名称、数量、单位、单价、小计）

#### Scenario: 查看已结账订单的关联记账记录
- **WHEN** 用户查看已结账订单详情
- **THEN** Dialog 展示关联的记账记录信息（金额、类型、渠道）

#### Scenario: 待结账订单的操作按钮
- **WHEN** 用户查看 Pending 状态订单详情
- **THEN** Dialog 显示"结账"、"编辑"、"取消"操作按钮

#### Scenario: 已结账订单无操作按钮
- **WHEN** 用户查看 Settled 状态订单详情
- **THEN** Dialog 不显示结账、编辑、取消按钮

### Requirement: 创建订单对话框
系统 SHALL 提供创建订单对话框，支持选择订单类型（销售/采购）、选择客户（可选）、添加订单明细、备注。不包含支付渠道选择。

#### Scenario: 创建销售订单
- **WHEN** 用户选择"销售订单"，添加商品明细（从商品列表搜索选择），点击确认
- **THEN** 系统创建 Pending 状态的销售订单，刷新列表

#### Scenario: 选择客户
- **WHEN** 用户在创建对话框中使用客户选择器选择客户
- **THEN** 系统将 customer_id 关联到订单

#### Scenario: 不选择客户（散客）
- **WHEN** 用户不选择客户直接创建订单
- **THEN** 系统创建散客订单，customer_id 为 None

#### Scenario: 添加订单明细行
- **WHEN** 用户在创建对话框中点击"添加明细"
- **THEN** 系统新增一行空白明细，用户可搜索选择商品或手动输入商品名称
- **THEN** 选择商品后自动填充单位和参考价格（可修改价格）
- **THEN** 计量单位从商品信息自动填充，不可手动修改

#### Scenario: 商品搜索覆盖多字段
- **WHEN** 用户在明细行中搜索商品
- **THEN** 搜索结果匹配商品名称、SKU 和关键词（keywords 字段）

#### Scenario: 自动计算订单总额
- **WHEN** 用户修改明细的数量或单价
- **THEN** 系统自动计算明细小计和订单应收总额

#### Scenario: 设置实收金额
- **WHEN** 用户在创建对话框中输入实收金额（默认等于应收总额）
- **THEN** 系统接受实收金额与应收金额不同（抹零/让利）

### Requirement: 编辑订单对话框
系统 SHALL 提供编辑订单对话框，允许修改 Pending 状态订单的明细和备注。

#### Scenario: 打开编辑对话框
- **WHEN** 用户在订单详情 Dialog 中点击"编辑"按钮
- **THEN** 弹出编辑对话框，预填充当前订单的明细和备注

#### Scenario: 修改订单明细
- **WHEN** 用户在编辑对话框中添加、删除或修改明细行后点击确认
- **THEN** 系统更新订单明细，自动重算应收和实收金额

#### Scenario: 编辑已结账订单
- **WHEN** 用户查看已结账订单
- **THEN** 详情 Dialog 不显示编辑按钮

### Requirement: 结账操作
系统 SHALL 在待结账订单的详情 Dialog 中提供"结账"操作，结账时选择支付渠道。

#### Scenario: 结账订单
- **WHEN** 用户在详情 Dialog 中点击"结账"按钮
- **THEN** 弹出结账确认对话框，显示订单摘要、支付渠道选择（必填）、实收金额（可修改）
- **WHEN** 用户选择支付渠道，确认结账
- **THEN** 系统将订单结账，自动生成记账记录
- **THEN** 订单状态变为"已结账"
- **THEN** 系统显示结账成功提示

#### Scenario: 结账未选择支付渠道
- **WHEN** 用户在结账确认对话框中未选择支付渠道就点击确认
- **THEN** 系统提示"请选择支付渠道"

### Requirement: 取消订单操作
系统 SHALL 在待结账订单的详情 Dialog 中提供"取消"操作，需二次确认。

#### Scenario: 取消订单
- **WHEN** 用户在详情 Dialog 中点击"取消"按钮，在确认对话框中点击确认
- **THEN** 系统将订单标记为已取消
- **THEN** 订单不可再结账或编辑

#### Scenario: 取消已结账订单
- **WHEN** 用户查看已结账订单
- **THEN** 详情 Dialog 不显示取消按钮

### Requirement: 导航入口
系统 SHALL 在侧边栏导航中添加"订单管理"入口。

#### Scenario: 侧边栏显示订单入口
- **WHEN** 应用侧边栏渲染时
- **THEN** 显示"订单管理"导航项，点击后导航到 `/orders`
