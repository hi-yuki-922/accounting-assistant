## ADDED Requirements

### Requirement: 记账记录查询支持时间范围过滤
系统 SHALL 支持按记账时间范围过滤记账记录，允许用户指定开始时间和结束时间。

#### Scenario: 按时间范围查询记录
- **WHEN** 用户指定账本 ID 为 10000001，开始时间为 2025-01-01，结束时间为 2025-01-31
- **THEN** 系统返回该账本下记账时间在 2025-01-01 到 2025-01-31 之间的记录

#### Scenario: 仅指定开始时间
- **WHEN** 用户仅指定开始时间为 2025-01-01
- **THEN** 系统返回该账本下记账时间大于等于 2025-01-01 的记录

#### Scenario: 仅指定结束时间
- **WHEN** 用户仅指定结束时间为 2025-01-31
- **THEN** 系统返回该账本下记账时间小于等于 2025-01-31 的记录

#### Scenario: 不指定时间范围
- **WHEN** 用户不指定时间范围
- **THEN** 系统返回该账本下所有时间范围的记录

### Requirement: 记账记录查询支持记账类型过滤
系统 SHALL 支持按记账类型过滤记账记录，记账类型包括：收入、支出、投资收益、投资亏损。

#### Scenario: 按记账类型查询收入记录
- **WHEN** 用户指定账本 ID 为 10000001，记账类型为收入
- **THEN** 系统返回该账本下所有记账类型为收入的记录

#### Scenario: 按记账类型查询支出记录
- **WHEN** 用户指定账本 ID 为 10000001，记账类型为支出
- **THEN** 系统返回该账本下所有记账类型为支出的记录

#### Scenario: 不指定记账类型
- **WHEN** 用户不指定记账类型
- **THEN** 系统返回该账本下所有记账类型的记录

### Requirement: 记账记录查询支持记账渠道过滤
系统 SHALL 支持按记账渠道过滤记账记录，记账渠道包括：现金、支付宝、微信、银行卡、未知。

#### Scenario: 按记账渠道查询支付宝记录
- **WHEN** 用户指定账本 ID 为 10000001，记账渠道为支付宝
- **THEN** 系统返回该账本下所有记账渠道为支付宝的记录

#### Scenario: 按记账渠道查询微信记录
- **WHEN** 用户指定账本 ID 为 10000001，记账渠道为微信
- **THEN** 系统返回该账本下所有记账渠道为微信的记录

#### Scenario: 不指定记账渠道
- **WHEN** 用户不指定记账渠道
- **THEN** 系统返回该账本下所有记账渠道的记录

### Requirement: 记账记录查询支持记录状态过滤
系统 SHALL 支持按记录状态过滤记账记录，记录状态包括：待入账、已入账。

#### Scenario: 按记录状态查询待入账记录
- **WHEN** 用户指定账本 ID 为 10000001，记录状态为待入账
- **THEN** 系统返回该账本下所有记录状态为待入账的记录

#### Scenario: 按记录状态查询已入账记录
- **WHEN** 用户指定账本 ID 为 10000001，记录状态为已入账
- **THEN** 系统返回该账本下所有记录状态为已入账的记录

#### Scenario: 不指定记录状态
- **WHEN** 用户不指定记录状态
- **THEN** 系统返回该账本下所有记录状态的记录

### Requirement: 分页查询过滤条件组合
系统 SHALL 支持同时应用多个过滤条件，所有过滤条件之间为 AND 关系。

#### Scenario: 组合使用时间范围和记账类型
- **WHEN** 用户指定账本 ID 为 10000001，时间范围为 2025-01-01 到 2025-01-31，记账类型为收入
- **THEN** 系统返回该账本下在指定时间范围内且记账类型为收入的记录

#### Scenario: 组合使用所有过滤条件
- **WHEN** 用户指定账本 ID 为 10000001，时间范围为 2025-01-01 到 2025-01-31，记账类型为收入，记账渠道为支付宝，记录状态为已入账
- **THEN** 系统返回该账本下满足所有过滤条件的记录

### Requirement: 分页查询固定过滤条件
系统 SHALL 在分页查询记账记录时，始终应用固定过滤条件，只查询 `write_off_id` 为 NULL 的记录。

#### Scenario: 查询时不返回冲账关联记录
- **WHEN** 用户查询账本 ID 为 10000001 的记录
- **THEN** 系统只返回 `write_off_id` 为 NULL 的记录，不返回已经被冲账的记录

### Requirement: 分页查询返回关联记录数量
系统 SHALL 在分页查询返回的每条记账记录中，包含该记录的冲账关联记录数量。

#### Scenario: 返回记录包含关联记录数量
- **WHEN** 用户查询账本 ID 为 10000001 的记录，其中某条记录有 3 条冲账关联记录
- **THEN** 系统返回的数据中该条记录包含关联记录数量为 3

#### Scenario: 返回记录关联记录数量为零
- **WHEN** 用户查询账本 ID 为 10000001 的记录，其中某条记录没有冲账关联记录
- **THEN** 系统返回的数据中该条记录包含关联记录数量为 0
