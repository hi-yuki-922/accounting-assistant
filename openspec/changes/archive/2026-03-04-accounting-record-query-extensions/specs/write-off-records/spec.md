## ADDED Requirements

### Requirement: 根据记账记录 ID 查询冲账关联记录
系统 SHALL 支持根据指定的记账记录 ID，查询所有通过 `write_off_id` 字段关联到该记录的记账记录。

#### Scenario: 查询记账记录的关联记录
- **WHEN** 用户指定记账记录 ID 为 2025010100001
- **THEN** 系统返回所有 `write_off_id` 等于 2025010100001 的记账记录

#### Scenario: 记录无关联记录
- **WHEN** 用户指定记账记录 ID 为 2025010100001，但该记录没有任何冲账关联记录
- **THEN** 系统返回空列表

#### Scenario: 记录有多条关联记录
- **WHEN** 用户指定记账记录 ID 为 2025010100001，该记录有 3 条冲账关联记录
- **THEN** 系统返回包含 3 条记账记录的列表

### Requirement: 关联记录查询返回完整记录信息
系统 SHALL 在查询冲账关联记录时，返回每条记录的完整信息，包括所有字段。

#### Scenario: 返回关联记录的完整信息
- **WHEN** 用户查询记账记录 ID 为 2025010100001 的关联记录
- **THEN** 系统返回每条关联记录的完整信息，包括 ID、金额、记账时间、记账类型、标题、记账渠道、备注、冲账 ID、创建时间、记录状态、账本 ID 等所有字段

### Requirement: 关联记录查询按创建时间倒序排列
系统 SHALL 在查询冲账关联记录时，按创建时间倒序排列返回结果。

#### Scenario: 关联记录按创建时间倒序
- **WHEN** 用户查询记账记录 ID 为 2025010100001 的关联记录
- **THEN** 系统返回的记录列表按 `create_at` 字段倒序排列，最新创建的记录排在最前面

### Requirement: 关联记录查询验证源记录存在
系统 SHALL 在查询冲账关联记录时，不验证源记录 ID 是否存在，直接根据 `write_off_id` 字段查询。

#### Scenario: 查询不存在的记录 ID
- **WHEN** 用户指定一个不存在的记账记录 ID 9999999999999
- **THEN** 系统返回空列表，不报错
